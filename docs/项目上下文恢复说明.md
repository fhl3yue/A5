# 项目上下文恢复说明

本文档用于 Codex 上下文窗口压缩、开启新会话或交接队友时快速恢复项目状态。

## 1. 项目定位

项目题目方向为“景区导览服务 AI 数字人”。当前实现不是通用闲聊数字人，而是面向景区可配置知识库的导览服务平台。

当前演示景区为：灵山胜境。

核心表达建议：

- 标题保留为“景区导览 AI 数字人”。
- 页面显示“当前景区：灵山胜境”。
- 快捷问题来自当前景区知识库推荐问题。
- 管理后台支持切换或配置景区资料与数字人形象。

## 4. 数字人形象管理状态

题目要求中“数字人形象管理：可配置数字人的外观、服装、声音等，使其更贴合景区文化特色”更偏向管理后台能力，不是游客自由换装。

当前实现方向：

- 管理者在后台配置数字人。
- 游客端读取当前景区数字人配置并展示。
- 配置项包括名称、角色标题、服务景区、服饰主题、授权角色素材模式、声音、欢迎语。
- 后端保存数字人配置。
- 文本问答和语音问答的 TTS 会使用后台配置的声音。
- 如果取得第三方角色授权，可将透明 PNG 放置为 `D:\software\frontend\assets\avatar\licensed-character.png`，并在后台服装主题中选择“授权角色素材”。

## 5. 当前限制与自纠结论

已经实现：

- 文本输入。
- 语音输入。
- 数字人语音回答。
- 表情变化。
- 播放状态下的口型动画。
- 后台数字人基础配置。

仍需如实说明的限制：

- 当前数字人是原创 SVG/CSS 分层动效角色，已升级为类 Live2D 虚拟偶像风格，但不是 Live2D Cubism SDK 模型。
- 项目不内置初音未来或其他第三方 IP 素材；如需使用，必须先取得权利方授权并自行提供授权素材文件。
- 当前口型同步是“语音播放状态驱动的嘴部动画”，不是逐音素级精准口型同步。
- 管理员鉴权仍是演示 token。
- 管理后台已补知识库管理、游客感受度报告和基础趋势看板，但仍可继续美化为更强的数据大屏。

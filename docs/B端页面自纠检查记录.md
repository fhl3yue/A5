# B端页面自纠检查记录

## 1. 检查目的

本记录用于说明当前 B 端页面在补齐前端与数字人展示后，是否偏离项目页面要求、演示限制与工程实现边界，并记录已经完成的修正。

## 2. 页面要求对照

### 2.1 游客端要求

当前已覆盖：

1. 数字人展示区
2. 文本问答
3. 语音录制与音频上传
4. 问答结果展示
5. 语音播报
6. 满意度反馈
7. 路线推荐
8. 参考来源展示
9. 当前示范景区提示
10. 自定义兴趣和自定义时长输入

结论：游客端主流程未偏离要求，已经能够形成演示闭环。

### 2.2 管理端要求

当前已覆盖：

1. 管理员登录
2. 数据看板
3. 热门问题统计
4. 情绪分布统计
5. 问答日志
6. 知识文档上传
7. 当前景区资料配置展示
8. 数字人形象配置
9. 数字人声音配置

结论：管理端基础能力未偏离要求，能够支撑比赛展示。

## 3. 数字人形象检查

当前数字人采用原创 SVG/CSS 分层动效角色，不使用第三方图片素材，也未直接复刻现有虚拟偶像或虚拟桌宠形象。最新版本已从“桌宠式角色”升级为“类 Live2D 虚拟导览员”观感。

已实现：

1. 完整虚拟导览员头身
2. 大眼、发丝、半透明衣袖等虚拟偶像化视觉特征
3. 分层头发、头部、脸部、身体、衣袖、手臂和配饰
4. 状态气泡
5. 待机漂浮
6. 眨眼动画
7. 发丝和衣袖摆动
8. 悬停挥手
9. 播报时口型动画
10. 语音/思考/聆听状态联动
11. 后台配置名称、身份、示范景区
12. 后台配置服装主题
13. 后台配置语音声线
14. 后台配置欢迎语
15. 后台支持“授权角色素材”展示模式，用于后续接入已获授权的第三方角色立绘

结论：当前版本属于原创类 Live2D 数字人演示方案，不是 3D 数字人，也不是 Live2D Cubism SDK 模型，但符合两人参赛项目的可控实现路线。

## 4. 已发现并修正的问题

### 4.1 旧角色样式残留

问题：从圆脸占位角色切换到 SVG 角色后，CSS 中仍残留旧角色的死代码。

处理：已清理旧角色相关样式和无用动画，降低维护混乱风险。

### 4.2 演示提示文案过强

问题：页面中存在明显偏“开发提示”的文字，例如录音建议、默认演示账号说明。

处理：已将相关文案调整为更像产品状态的表达，减少演示痕迹。

### 4.3 移动端角色溢出

问题：移动端截图中角色和气泡在窄屏下存在靠右裁切风险。

处理：已调整移动端角色尺寸、气泡定位和页面横向溢出控制。

### 4.4 动效可访问性

问题：数字人有持续动画，如果用户系统设置减少动态效果，页面应减少动画。

处理：已加入 `prefers-reduced-motion` 适配。

## 5. 当前仍需说明的边界

1. 当前数字人为 SVG/CSS 类 Live2D 分层动效角色，不是 3D 建模角色，也不是 Live2D Cubism SDK 模型。
2. 口型动画是根据音频播放状态联动，不是逐字级精准唇形同步。
3. 管理端登录仍是演示 token，不是正式权限系统。
4. 图表目前为轻量 CSS 展示，没有接入 ECharts。
5. 语音识别效果受本机麦克风、浏览器权限和 ASR 模型影响。
6. 当前支持通过知识文档补充景区资料，但还没有完整多景区切换表单。
7. 项目不内置初音未来或其他第三方 IP 素材；若使用第三方虚拟偶像形象，需要先取得授权并提供本地授权素材。
